\section{Описание набора данных}

\subsection{Источник данных}

Основной набор данных — \textit{Skin Burn Dataset}, опубликованный на платформе Kaggle~\cite{kaggle_burn_dataset}, содержащий \textbf{1\,225 изображений} ожоговых поражений кожи с аннотациями в формате YOLO. Датасет включает три класса ожогов различной степени тяжести с размеченными ограничивающими прямоугольниками (bounding boxes).

\section{Формат аннотаций YOLO}

Каждое изображение сопровождается текстовым файлом аннотации в формате YOLO:

\begin{equation}
    \text{annotation} = (\texttt{class\_id}, x_c, y_c, w, h)
\label{eq:yolo_format}
\end{equation}

\noindent где \texttt{class\_id} — идентификатор класса (0, 1 или 2); $(x_c, y_c)$ — нормализованные координаты центра ограничивающего прямоугольника; $(w, h)$ — нормализованные ширина и высота. Все значения находятся в диапазоне $[0, 1]$, нормализованы относительно размеров изображения.

Пример аннотации:
\begin{verbatim}
1 0.5717 0.5138 0.2975 0.4309
1 0.5860 0.1519 0.1541 0.1823
\end{verbatim}

Это означает, что на изображении обнаружены два ожога второй степени (class\_id = 1) с указанными координатами.

Датасет разделяется на обучающую и валидационную выборки с соотношением 80/20 и фиксированным random seed для воспроизводимости:

\begin{lstlisting}[language=Python, caption=Разделение данных, label=lst:split]
import random

random.seed(42)

image_files = list(source_dir.glob("*.jpg"))
random.shuffle(image_files)

split_idx = int(len(image_files) * (1 - VAL_SPLIT))
train_images = image_files[:split_idx]
val_images = image_files[split_idx:]
\end{lstlisting}

\section{Архитектура модели YOLOv8n}

\subsection{Выбор варианта модели}

Для данного исследования выбран YOLOv8n (nano) — компактный вариант архитектуры с приблизительно 3,2~млн параметров. Выбор обоснован следующими факторами:

\begin{enumerate}
    \item \textbf{Баланс скорости и точности}: YOLOv8n обеспечивает детекцию в реальном времени при сохранении конкурентной точности.
    \item \textbf{Размер датасета}: Для датасета из 1\,225 изображений крупные модели избыточны и склонны к переобучению.
    \item \textbf{Практическое применение}: Компактная модель легче интегрировать в мобильные и веб-приложения.
\end{enumerate}

\subsection{Архитектурные компоненты}

\textbf{Backbone (CSPDarknet):} Извлекает признаки из входного изображения через последовательность C2f-блоков (Cross Stage Partial с двумя свёртками):

\begin{equation}
    \mathbf{h}_{out} = \text{Concat}\left(\mathbf{h}_{main}, \mathbf{h}_{shortcut}\right)
\label{eq:c2f}
\end{equation}

\noindent где $\mathbf{h}_{main}$ — выход основной ветви через bottleneck-блоки; $\mathbf{h}_{shortcut}$ — shortcut-соединение.

\textbf{Neck (PANet):} Path Aggregation Network агрегирует признаки разных масштабов для детекции объектов различного размера.

\textbf{Head (Decoupled):} Разделённая голова с независимыми ветвями для классификации и регрессии координат.

\section{Фреймворк оценки}

\subsection{Метрики детекции объектов}

\textbf{Precision (точность)} — доля правильных детекций:

\begin{equation}
    P = \frac{TP}{TP + FP}
\label{eq:precision_det}
\end{equation}

\textbf{Recall (полнота)} — доля обнаруженных объектов:

\begin{equation}
    R = \frac{TP}{TP + FN}
\label{eq:recall_det}
\end{equation}

\textbf{IoU (Intersection over Union):}

\begin{equation}
    \text{IoU} = \frac{|B_{pred} \cap B_{gt}|}{|B_{pred} \cup B_{gt}|}
\label{eq:iou_metric}
\end{equation}

\textbf{AP (Average Precision)} — площадь под кривой precision-recall:

\begin{equation}
    \text{AP} = \int_0^1 P(R) \, dR
\label{eq:ap}
\end{equation}

\textbf{mAP (mean Average Precision):}

\begin{equation}
    \text{mAP} = \frac{1}{|\mathcal{C}|} \sum_{c \in \mathcal{C}} \text{AP}_c
\label{eq:map_metric}
\end{equation}

\subsection{Пороги IoU}

Стандартные метрики COCO:
\begin{itemize}
    \item \textbf{mAP@0.5} — порог IoU = 0.5
    \item \textbf{mAP@0.5:0.95} — среднее по порогам от 0.5 до 0.95 с шагом 0.05
\end{itemize}

\begin{lstlisting}[language=Python, caption=Обучение модели YOLOv8, label=lst:train]
from ultralytics import YOLO
from pathlib import Path

DATASET_DIR = Path("dataset")
MODELS_DIR = Path("models")

def train(
    epochs: int = 100,
    batch: int = 32,
    imgsz: int = 640,
    model_size: str = "n"
) -> Path:
    model = YOLO(f"yolov8{model_size}.pt")

    results = model.train(
        data=str(DATASET_DIR / "data.yaml"),
        epochs=epochs,
        imgsz=imgsz,
        batch=batch,
        project=str(MODELS_DIR),
        name="burn_detector",
        exist_ok=True,
    )

    best_model = MODELS_DIR / "burn_detector" / "weights" / "best.pt"
    return best_model
\end{lstlisting}
