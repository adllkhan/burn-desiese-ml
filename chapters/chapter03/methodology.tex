\section{Описание набора данных}

\subsection{Источник данных}
В работе используется набор данных \textit{Skin Burn Dataset}, опубликованный на Kaggle~\cite{kaggle_burn_dataset}. Датасет содержит \textbf{1\,225 изображений} ожогов кожи. Каждое изображение снабжено аннотациями в формате YOLO. Ожоги распределены на три класса по степени тяжести; для каждого изображения указаны ограничивающие прямоугольники (bounding boxes).

\subsection{Описание классов}

\section{Формат аннотаций YOLO}

Для каждого изображения создаётся отдельный файл аннотации в формате YOLO:

\begin{equation}
\text{annotation} = (\texttt{class\_id},\, x_c,\, y_c,\, w,\, h)
\label{eq:yolo_format}
\end{equation}

\noindent где \texttt{class\_id} --- номер класса (0, 1 или 2); $(x_c, y_c)$ --- нормированные координаты центра прямоугольника; $(w, h)$ --- нормированные ширина и высота. Все числа лежат в диапазоне $[0,1]$ относительно размеров изображения.

Пример аннотации (два ожога второй степени на одном изображении):

\begin{verbatim}
1 0.5717 0.5138 0.2975 0.4309
1 0.5860 0.1519 0.1541 0.1823
\end{verbatim}

\section{Архитектура модели YOLOv8n}

\subsection{Выбор варианта модели}

Для исследования выбрана модель YOLOv8n (nano) --- наименьший вариант линейки YOLOv8, содержащий около 3{,}2~млн параметров. Выбор обусловлен следующими причинами:

\begin{enumerate}
\item \textbf{Баланс скорости и точности.} YOLOv8n обеспечивает быструю детекцию объектов при сохранении высокой точности, что подходит для применения в режиме реального времени.
\item \textbf{Размер датасета.} При наличии 1\,225 изображений более крупные модели склонны к переобучению; компактная модель является более надёжным выбором.
\item \textbf{Практическое применение.} Компактную модель проще интегрировать в мобильные и веб-приложения.
\end{enumerate}

\subsection{Архитектурные компоненты}

\textbf{Backbone (CSPDarknet).} Извлекает признаки из входного изображения, последовательно применяя свёрточные блоки C2f (Cross Stage Partial из двух свёрток) с остаточными соединениями:

\begin{equation}
\mathbf{h}_{out} = \text{Concat}(\mathbf{h}_{main},\, \mathbf{h}_{shortcut})
\label{eq:c2f}
\end{equation}

\noindent где $\mathbf{h}_{main}$ --- выход основной ветви, проходящей через bottleneck-блоки; $\mathbf{h}_{shortcut}$ --- shortcut-соединение.

\textbf{Neck (PANet).} Path Aggregation Network агрегирует признаки разных масштабов, что позволяет детектировать объекты разных размеров.

\textbf{Head (Decoupled).} Голова с двумя независимыми ветвями: одна выполняет классификацию, другая --- регрессию координат.

\section{Фреймворк оценки}

\subsection{Метрики детекции объектов}

\textbf{Precision (точность)} --- доля правильных детекций:

\begin{equation}
    P = \frac{TP}{TP + FP}
\label{eq:precision_det}
\end{equation}

\textbf{Recall (полнота)} --- доля обнаруженных объектов:

\begin{equation}
    R = \frac{TP}{TP + FN}
\label{eq:recall_det}
\end{equation}

\textbf{IoU (Intersection over Union):}

\begin{equation}
    \text{IoU} = \frac{|B_{pred} \cap B_{gt}|}{|B_{pred} \cup B_{gt}|}
\label{eq:iou_metric}
\end{equation}

\textbf{AP (Average Precision)} --- площадь под кривой precision-recall:

\begin{equation}
    \text{AP} = \int_0^1 P(R) \, dR
\label{eq:ap}
\end{equation}

\textbf{mAP (mean Average Precision):}

\begin{equation}
    \text{mAP} = \frac{1}{|\mathcal{C}|} \sum_{c \in \mathcal{C}} \text{AP}_c
\label{eq:map_metric}
\end{equation}

\subsection{Пороги IoU}

Стандартные метрики COCO:
\begin{itemize}
    \item \textbf{mAP@0.5} --- порог IoU = 0.5
    \item \textbf{mAP@0.5:0.95} --- среднее по порогам от 0.5 до 0.95 с шагом 0.05
\end{itemize}

\begin{lstlisting}[language=Python, caption=Обучение модели YOLOv8, label=lst:train]
from ultralytics import YOLO
from pathlib import Path

DATASET_DIR = Path("dataset")
MODELS_DIR = Path("models")

def train(
    epochs: int = 100,
    batch: int = 32,
    imgsz: int = 640,
    model_size: str = "n"
) -> Path:
    model = YOLO(f"yolov8{model_size}.pt")

    results = model.train(
        data=str(DATASET_DIR / "data.yaml"),
        epochs=epochs,
        imgsz=imgsz,
        batch=batch,
        project=str(MODELS_DIR),
        name="burn_detector",
        exist_ok=True,
    )

    best_model = MODELS_DIR / "burn_detector" / "weights" / "best.pt"
    return best_model
\end{lstlisting}
