\section{Описание набора данных}

\subsection{Источник данных}
В работе используется набор данных \textit{Skin Burn Dataset}, опубликованный на платформе Kaggle~\cite{kaggle_burn_dataset}. Выбор данного датасета обусловлен тремя факторами. Во-первых, он является одним из немногих общедоступных наборов медицинских изображений с разметкой по степеням ожогов, что критически важно для воспроизводимости результатов и сопоставимости с другими исследованиями~\cite{chauhan2020burn,khan2021burn}. Во-вторых, изображения снабжены аннотациями в формате YOLO с указанием ограничивающих прямоугольников (bounding boxes), что позволяет решать одновременно задачу детекции и классификации. В-третьих, разнообразие условий съёмки (различные ракурсы, освещение, цвет кожи, локализация ожога) обеспечивает близость распределения данных к реальной клинической практике.

Датасет содержит \textbf{1\,225 изображений} ожогов кожи в формате JPEG. Ожоги распределены на три класса по степени тяжести в соответствии с общепринятой клинической классификацией~\cite{peck2012burns}. Каждое изображение снабжено отдельным текстовым файлом аннотации, содержащим координаты ограничивающих прямоугольников и идентификаторы классов. Среднее число аннотаций на одно изображение составляет 1,81, что отражает реальную клиническую ситуацию: на одной фотографии может присутствовать несколько ожоговых поражений различной степени.

Несмотря на достоинства, датасет имеет ряд ограничений. Объём в 1\,225 изображений считается умеренным для задач глубокого обучения; для сравнения, классический датасет COCO~\cite{lin2014coco} содержит более 330\,000 изображений с 80 классами, а ImageNet --- более 14 миллионов. Этот фактор накладывает ограничения на выбор архитектуры модели (см.~раздел~\ref{sec:model_choice}) и определяет необходимость применения трансферного обучения и интенсивной аугментации данных.

\subsection{Распределение классов и разбиение выборки}
\label{sec:split}

Для обучения и оценки модели набор данных разделён в соотношении 80\,\%~/~20\,\%: 80\,\% ($n{=}980$) --- обучающая выборка, 20\,\% ($n{=}245$) --- валидационная. Указанное соотношение является одним из наиболее распространённых для задач компьютерного зрения~\cite{he2016resnet,everingham2010pascal} и обеспечивает компромисс между объёмом данных, доступных для обучения, и статистической надёжностью оценки качества модели на валидации. Разбиение выполнено со стратификацией по классам и фиксированным значением случайного зерна (\texttt{random.seed(42)}), что гарантирует воспроизводимость результатов и согласованность между независимыми запусками.

В датасете наблюдается выраженный дисбаланс классов: ожоги третьей степени встречаются примерно в 2{,}8 раза реже, чем ожоги второй степени, и в 2{,}5 раза реже, чем ожоги первой степени. Это согласуется с клинической эпидемиологией~\cite{peck2012burns}: тяжёлые ожоги III степени действительно представляют меньшую долю общего потока пациентов, поскольку чаще приводят к госпитализации в специализированные ожоговые центры. Тем не менее с точки зрения обучения модели дисбаланс представляет проблему: при отсутствии корректирующих мер модель может склоняться к преобладающему классу, занижая чувствительность к редким, но клинически наиболее значимым случаям. Для частичной компенсации этого эффекта в работе применяется внутренний механизм балансировки выборок YOLOv8 на основе функции потерь с учётом частот классов, а также интенсивная аугментация данных.

\section{Формат аннотаций YOLO}

Формат аннотаций YOLO был выбран в качестве промышленного стандарта для задач детекции в реальном времени~\cite{redmon2016yolo,ultralytics2023yolov8}. Его основное преимущество --- компактность представления и нормированность координат, что упрощает работу с изображениями разного разрешения без необходимости дополнительного масштабирования.

Для каждого изображения создаётся отдельный текстовый файл аннотации с расширением \texttt{.txt}, имя которого совпадает с именем изображения. Каждая строка файла соответствует одной аннотации и имеет следующий формат:

\begin{equation}
\text{annotation} = (\texttt{class\_id},\, x_c,\, y_c,\, w,\, h)
\label{eq:yolo_format}
\end{equation}

Формула~\eqref{eq:yolo_format} описывает структуру одной строки аннотации, где \texttt{class\_id} --- целочисленный идентификатор класса (0, 1 или 2 в соответствии с Таблицей~\ref{tab:classes}); $(x_c, y_c)$ --- нормированные координаты центра ограничивающего прямоугольника; $(w, h)$ --- нормированные ширина и высота прямоугольника. Все четыре геометрических значения принадлежат диапазону $[0,1]$ и вычисляются как отношения к соответствующим размерам изображения. Такая нормировка делает аннотацию инвариантной к разрешению, что удобно при работе с разнородными источниками.

Пример файла аннотации, описывающего два ожога второй степени на одном изображении:

\begin{verbatim}
1 0.5717 0.5138 0.2975 0.4309
1 0.5860 0.1519 0.1541 0.1823
\end{verbatim}

Каждая строка соответствует независимому ограничивающему прямоугольнику; YOLO-формат не накладывает ограничений на максимальное число аннотаций в одном файле, что делает его пригодным для изображений с несколькими ожоговыми поражениями.

\section{Конвейер предобработки данных}

\subsection{Загрузка и подготовка датасета}

Загрузка датасета автоматизирована с помощью библиотеки \texttt{kagglehub}, что позволяет получать данные напрямую с Kaggle, без необходимости ручного скачивания. Такой подход обеспечивает воспроизводимость: версия датасета фиксируется идентификатором, а скрипт загрузки может быть выполнен на любой машине с установленным окружением Python.

\begin{lstlisting}[language=Python, caption=Загрузка датасета из Kaggle с использованием \texttt{kagglehub}, label=lst:download]
import kagglehub
from pathlib import Path

def download_dataset() -> Path:
    path = kagglehub.dataset_download(
        "shubhambaid/skin-burn-dataset"
    )
    return Path(path)
\end{lstlisting}

Функция \texttt{download\_dataset()} возвращает абсолютный путь к корневому каталогу датасета, что используется на последующих этапах подготовки. При повторных вызовах библиотека использует локальный кэш и не выполняет повторную загрузку.

\subsection{Разделение на train/val}

Разделение датасета на обучающую и валидационную части выполняется с использованием фиксированного random seed, что является общепринятой практикой для обеспечения воспроизводимости в задачах глубокого обучения~\cite{he2016resnet}. Соотношение 80\,\%/20\,\% обосновано в разделе~\ref{sec:split}.

\begin{lstlisting}[language=Python, caption=Разделение данных на обучающую и валидационную выборки, label=lst:split]
import random

VAL_SPLIT = 0.2
random.seed(42)

image_files = list(source_dir.glob("*.jpg"))
random.shuffle(image_files)

split_idx = int(len(image_files) * (1 - VAL_SPLIT))
train_images = image_files[:split_idx]
val_images = image_files[split_idx:]
\end{lstlisting}

После случайного перемешивания списка файлов с фиксированным зерном (\texttt{seed=42}) первые 980 изображений отбираются в обучающую выборку, а оставшиеся 245 --- в валидационную. Соответствующие файлы аннотаций копируются в одноимённые подкаталоги \texttt{labels/train} и \texttt{labels/val}.

\subsection{Формирование структуры директорий}

Для совместимости с YOLOv8 формируется стандартизированная структура каталогов, ожидаемая фреймворком Ultralytics~\cite{ultralytics2023yolov8}:

\begin{verbatim}
dataset/
|-- images/
|   |-- train/   # 980 изображений
|   |-- val/     # 245 изображений
|-- labels/
|   |-- train/   # 980 файлов аннотаций
|   |-- val/     # 245 файлов аннотаций
|-- data.yaml    # Конфигурационный файл
\end{verbatim}

Подобная структура позволяет фреймворку автоматически сопоставлять изображения и аннотации по совпадению имён файлов и не требует дополнительных индексных файлов.

\subsection{Конфигурационный файл data.yaml}

Конфигурационный файл \texttt{data.yaml} описывает пути к данным и имена классов. Этот файл является единственным источником информации о датасете, передаваемым в \texttt{model.train()}.

  1: second_degree
  2: third_degree
\end{lstlisting}

\section{Архитектура модели YOLOv8n}
\label{sec:model_choice}

\subsection{Выбор варианта модели}

Линейка YOLOv8 включает пять вариантов модели, различающихся числом параметров и вычислительной сложностью: n (nano), s (small), m (medium), l (large) и x (extra-large)~\cite{ultralytics2023yolov8}. Число параметров варьируется от 3{,}2 млн у YOLOv8n до 68{,}2 млн у YOLOv8x. Для данной работы выбран вариант \textbf{YOLOv8n} --- наименьший в линейке, содержащий около 3{,}2 млн обучаемых параметров. Выбор обусловлен следующими соображениями:

\begin{enumerate}
\item Баланс скорости и точности. YOLOv8n обеспечивает скорость инференса свыше 20 FPS даже на процессорах общего назначения, сохраняя при этом качество детекции, сопоставимое с более крупными моделями на задачах с ограниченным числом классов.
\item Размер датасета. При наличии всего 1\,225 изображений более крупные модели (YOLOv8m, YOLOv8l, YOLOv8x) склонны к переобучению, что подтверждается результатами предварительных экспериментов и согласуется с общими наблюдениями в задачах с ограниченным объёмом данных~\cite{shorten2019augmentation}.
\item Практическое применение. Компактную модель размером 6{,}2~МБ проще интегрировать в мобильные и веб-приложения, а также развёртывать на устройствах с ограниченными вычислительными ресурсами, что является важным фактором для применения в условиях ограниченной инфраструктуры~\cite{vastai2023}.
\end{enumerate}

\subsection{Архитектурные компоненты}

Архитектура YOLOv8 состоит из трёх основных блоков: backbone (выделение признаков), neck (агрегация признаков) и head (предсказание боксов и классов). Рассмотрим каждый из них подробнее.

\textbf{Backbone (CSPDarknet).} Извлекает иерархические признаки из входного изображения, последовательно применяя свёрточные блоки C2f (Cross Stage Partial из двух свёрток) с остаточными соединениями. Архитектура CSPDarknet является развитием идей, заложенных в Darknet-53 в YOLOv3~\cite{redmon2018yolov3} и доработанных в YOLOv4~\cite{bochkovskiy2020yolov4}. C2f-блок выполняет следующее преобразование:

\begin{equation}
\mathbf{h}_{out} = \text{Concat}(\mathbf{h}_{main},\, \mathbf{h}_{shortcut})
\label{eq:c2f}
\end{equation}

Уравнение~\eqref{eq:c2f} описывает агрегацию двух потоков признаков внутри C2f-блока, где $\mathbf{h}_{main}$ --- выход основной ветви, проходящей через bottleneck-блоки; $\mathbf{h}_{shortcut}$ --- shortcut-соединение, передающее признаки в обход bottleneck-преобразований. Операция $\text{Concat}$ выполняет конкатенацию вдоль канальной оси. Подобное соединение позволяет одновременно сохранять низкоуровневые признаки (через shortcut) и обогащать представление высокоуровневыми (через bottleneck), что положительно сказывается на качестве детекции мелких объектов.

\textbf{Neck (PANet).} Path Aggregation Network агрегирует признаки разных масштабов, формируя многомасштабное представление, что позволяет детектировать объекты разных размеров. Идея PANet развивает Feature Pyramid Network (FPN) из YOLOv3~\cite{redmon2018yolov3}, добавляя нисходящий поток признаков от мелких масштабов к крупным.

\textbf{Head (Decoupled Head).} В отличие от ранних версий YOLO с объединённой головой, YOLOv8 использует декомпозированную голову с двумя независимыми ветвями: одна выполняет классификацию, другая --- регрессию координат ограничивающего прямоугольника. Кроме того, YOLOv8 является безъякорной (anchor-free) архитектурой, что упрощает настройку модели на новые данные и устраняет необходимость в подборе предопределённых якорей.

\subsection{Трансферное обучение}

Обучение модели с нуля на 1\,225 изображениях не позволило бы достичь приемлемой обобщающей способности. По этой причине применяется трансферное обучение~\cite{yosinski2014transfer}: модель инициализируется весами, предобученными на крупномасштабном датасете COCO~\cite{lin2014coco}, содержащем 80 классов общих объектов и более 330\,000 изображений. На этапе предобучения backbone и neck формируют универсальные представления о визуальных признаках (краях, текстурах, формах), которые впоследствии адаптируются под специфическую задачу классификации ожогов.

Веса загружаются из файла \texttt{yolov8n.pt}, поставляемого вместе с фреймворком Ultralytics:

\begin{lstlisting}[language=Python, caption=Инициализация модели YOLOv8n с предобученными на COCO весами, label=lst:init]
from ultralytics import YOLO

model = YOLO("yolov8n.pt")  # 6.2 MB
\end{lstlisting}

В отличие от классических подходов с замораживанием нижних слоёв~\cite{yosinski2014transfer}, в YOLOv8 рекомендуется выполнять полное дообучение всех слоёв с пониженной начальной скоростью обучения, что согласуется с современной практикой работы с компактными моделями.

\subsection{Весовые коэффициенты потерь}

Функция потерь YOLOv8 представляет собой взвешенную сумму трёх компонент: регрессии бокса, классификации и distribution focal loss. Полная функция имеет вид:

\begin{equation}
\mathcal{L} = \lambda_{box}\,\mathcal{L}_{box} + \lambda_{cls}\,\mathcal{L}_{cls} + \lambda_{dfl}\,\mathcal{L}_{dfl}
\label{eq:total_loss}
\end{equation}

Уравнение~\eqref{eq:total_loss} объединяет три компонента: $\mathcal{L}_{box}$ --- CIoU loss для регрессии ограничивающих прямоугольников, штрафующий за расхождение предсказанного и истинного боксов с учётом IoU, расстояния между центрами и соотношения сторон; $\mathcal{L}_{cls}$ --- бинарная кросс-энтропия (BCE) для классификации, применяемая независимо к каждому классу; $\mathcal{L}_{dfl}$ --- Distribution Focal Loss, позволяющий моделировать координаты как распределение, что улучшает точность локализации. Весовые коэффициенты $\lambda_{box}$, $\lambda_{cls}$, $\lambda_{dfl}$ заданы по умолчанию (Таблица~\ref{tab:loss_weights}).

\section{Фреймворк оценки}

Для оценки качества модели применяется стандартный набор метрик задачи детекции объектов, утвердившийся в литературе благодаря таким бенчмаркам, как PASCAL VOC~\cite{everingham2010pascal} и COCO~\cite{lin2014coco}.

\subsection{Метрики детекции объектов}

\textbf{Precision (точность).} Доля правильных детекций среди всех предсказанных:

\begin{equation}
    P = \frac{TP}{TP + FP}
\label{eq:precision_det}
\end{equation}

Формула~\eqref{eq:precision_det} использует обозначения: $TP$ (true positive) --- число корректно детектированных объектов; $FP$ (false positive) --- число ложных детекций. Высокий precision означает, что модель редко делает ложные срабатывания, что критически важно в клинических приложениях.

\textbf{Recall (полнота).} Доля обнаруженных объектов среди всех существующих:

\begin{equation}
    R = \frac{TP}{TP + FN}
\label{eq:recall_det}
\end{equation}

В уравнении~\eqref{eq:recall_det} $FN$ (false negative) --- число пропущенных объектов. Высокий recall особенно важен в задачах медицинского скрининга: пропуск ожогового поражения может привести к ошибочной тактике лечения.

\textbf{IoU (Intersection over Union).} Мера степени перекрытия предсказанного и истинного боксов:

\begin{equation}
    \text{IoU} = \frac{|B_{pred} \cap B_{gt}|}{|B_{pred} \cup B_{gt}|}
\label{eq:iou_metric}
\end{equation}

Уравнение~\eqref{eq:iou_metric} использует $B_{pred}$ для предсказанного бокса и $B_{gt}$ --- для ground truth. IoU принимает значения от 0 (нет перекрытия) до 1 (полное совпадение). Детектируемый объект считается true positive, если $\text{IoU} \geq \text{threshold}$ для выбранного порога (обычно 0{,}5).

\textbf{AP (Average Precision).} Площадь под кривой precision-recall:

\begin{equation}
    \text{AP} = \int_0^1 P(R)\,dR
\label{eq:ap}
\end{equation}

Формула~\eqref{eq:ap} даёт скалярную меру качества, объединяющую precision и recall на всём диапазоне порогов уверенности. AP вычисляется для каждого класса отдельно.

\textbf{mAP (mean Average Precision).} Среднее AP по всем классам:

\begin{equation}
    \text{mAP} = \frac{1}{|\mathcal{C}|} \sum_{c \in \mathcal{C}} \text{AP}_c
\label{eq:map_metric}
\end{equation}

В формуле~\eqref{eq:map_metric} $\mathcal{C}$ --- множество классов задачи (в данном случае $|\mathcal{C}|=3$); $\text{AP}_c$ --- средняя точность для класса $c$. mAP является основной метрикой при сравнении детекторов объектов и используется во всех современных бенчмарках~\cite{lin2014coco,everingham2010pascal}.

\subsection{Пороги IoU}

Для оценки качества детекции применяются два стандартных протокола, унаследованных от COCO~\cite{lin2014coco}:

\begin{itemize}
    \item mAP@0.5 --- mAP при фиксированном пороге $\text{IoU}=0{,}5$ (PASCAL VOC convention~\cite{everingham2010pascal});
    \item mAP@0.5:0.95 --- среднее mAP по десяти порогам IoU от 0{,}5 до 0{,}95 с шагом 0{,}05 (COCO convention).
\end{itemize}

Протокол mAP@0{,}5:0{,}95 строже и оценивает не только корректность класса, но и точность локализации, что особенно важно в медицинских приложениях, где точное выделение области поражения может использоваться для последующей оценки площади ожога (TBSA).

\begin{lstlisting}[language=Python, caption=Основной скрипт обучения модели YOLOv8, label=lst:train]
from ultralytics import YOLO
from pathlib import Path

DATASET_DIR = Path("dataset")
MODELS_DIR = Path("models")

def train(
    epochs: int = 100,
    batch: int = 32,
    imgsz: int = 640,
    model_size: str = "n"
) -> Path:
    model = YOLO(f"yolov8{model_size}.pt")

    results = model.train(
        data=str(DATASET_DIR / "data.yaml"),
        epochs=epochs,
        imgsz=imgsz,
        batch=batch,
        project=str(MODELS_DIR),
        name="burn_detector",
        exist_ok=True,
    )

    best_model = MODELS_DIR / "burn_detector" / "weights" / "best.pt"
    return best_model
\end{lstlisting}
